# Help Scout MCP Server v1.9.0

## Release Positioning

v1.9.0 is the read API parity release. It exposes the current Help Scout Mailbox and Docs read surfaces as MCP tools and adds the shared foundation needed for write-capable API parity work after this release.

No write tools are included in v1.9.0.

## Highlights

- Adds first-class List Customers v3 support through `listCustomersV3`.
- Completes read-only coverage for conversation detail, customer contact sub-resources, organization traversal, metadata, reports, and Docs.
- Adds Docs fixture seeding and live dogfood coverage for sites, collections, categories, articles, revisions, related articles, and redirects.
- Documents the MCP tool contract, MCP vs CLI boundary, and write-tool safety contract for the next API-parity phase.
- Refreshes npm, MCPB, Docker, registry, and navigator plugin metadata for the 1.9.0 release boundary.

## Compatibility

- Existing OAuth2 credentials continue to use `HELPSCOUT_APP_ID` and `HELPSCOUT_APP_SECRET`.
- `HELPSCOUT_DOCS_API_KEY` is optional and only required for Docs tools.
- Existing read tools remain available; this release is additive for the MCP tool surface.
- By default `tools/list` continues to advertise the full flat read catalog. Clients that want the smaller discovery/meta surface can opt in with `HELPSCOUT_TOOL_SURFACE=compact`.
- The navigator plugin pins `help-scout-mcp-server@1.9.0` so plugin users do not accidentally pull a later package version.
- `REDACT_MESSAGE_CONTENT=true` remains the supported way to hide message bodies in tool responses. It is not a compliance boundary.

## Release Gate

Before tagging and publishing:

- `npm run type-check`
- `npm run lint`
- `npm run build`
- `npm test -- --runInBand --silent`
- `npm pack --dry-run`
- package tarball install smoke
- `npm run mcpb:build`
- `npm run mcpb:pack`
- `npm run test:contract`
- `npm run test:docker:ci`
- `npm run test:docker`
- live dogfood seed with Docs credentials
- `npm run dogfood:full`
- `npm run security`
- `npm run security:full`

## Publish Sequence

1. Merge the `dev` release-prep PR into `main`.
2. Tag `v1.9.0` from the merge commit.
3. Publish `help-scout-mcp-server@1.9.0` to npm.
4. Build and attach the `.mcpb` artifact to the GitHub release.
5. Publish Docker `1.9.0` only when ready to move Docker consumers.
6. Verify public install paths:
   - `npx -y help-scout-mcp-server@1.9.0`
   - navigator plugin install
   - `.mcpb` install
   - Docker `drewburchfield/help-scout-mcp-server:1.9.0` if published
